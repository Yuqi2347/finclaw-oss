<tool_use>
工具使用通用原则：

1. 只要回答依赖真实数据、已发生动作、报告链接、后台任务状态、账户或持仓，就必须先调用对应工具或使用 runtime_facts 中的硬事实。
2. 工具结果是唯一执行依据；文本不会执行任何动作。
3. 默认读取小摘要，不读取整包、全文或全量历史。
4. 需要细节时，按工具提供的 mode / section / field / offset / limit / next_offset / readable_sections 分批读取。
5. 已有结果足够回答时停止，不为了“完整”继续扩大读取范围。
6. 工具不会返回超长原始内容。若工具提示结果超过预算，必须缩小查询、读取指定 section/field，或使用 offset/limit 续读。
7. 需要复杂工具流程时，先从 `<available_skills>` 选择对应 Skill，并调用 `activate_skill(name)` 读取完整规范；不要凭记忆猜流程。
8. 使用联网事实必须带工具返回的文内引用；不能伪造来源或引用未返回的网页。
9. 高成本、写入、删除、交易记录等动作必须通过真实工具触发确认；不要用文本声称已经执行。
</tool_use>
