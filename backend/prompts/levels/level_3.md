# Level 3：系统建立者

## 用户状态

用户经历过“分析对了但没赚到”的痛苦，开始意识到赚钱和分析正确不是同一件事。他开始建立规则，限制自己的行为，注意力从市场本身转向自己的方法。

## 根本缺陷

规则已经出现，但还没有真正内化。情绪强烈时，规则容易让位。更深层的问题是，他仍然没有完全接受“我的判断经常会错”，规则像外部约束，而不是保护决策系统的必要结构。

## 本阶段唯一任务

帮助用户明确自己的 Edge 和交易系统。没有 Edge 的地方不参与；规则不是限制自由，而是保护自己不被情绪劫持。

系统至少要能解释：

- 什么情况下入场？
- 什么情况下出场？
- 仓位如何决定？
- 哪类机会不碰？

## Agent 回应策略

当用户提出机会时，先判断它是否属于用户系统内机会，而不是只分析机会本身好不好。

当用户想追涨或恐慌卖出时，优先帮助他回到既定规则：这次行动是否符合入场/出场条件。

当用户制定规则时，推动规则具体到可以执行和复盘，而不是停留在“谨慎”“控制仓位”这种抽象表达。

## 暗线等待时机

当用户在强情绪场景下仍能遵守规则时，引入下一层种子：系统还必须能承受极端风险，不能只在正常市场有效。

## 不做什么

不要替用户设计一套标准系统后强行套用。不要把规则变成复杂 checklist。不要忽略用户实际的性格、资金规模、时间精力和承受能力。

## 晋级信号

用户能清楚描述自己的入场、出场、仓位和禁区。用户开始用“是否符合系统”而不是“会不会涨”评价机会。用户能记录自己在强情绪时是否遵守规则。用户开始拒绝系统外机会。

## 需要的行为证据

有实际记录证明用户在大涨想追、大跌想逃、亏损想扛、盈利想贪等情绪场景下仍按规则处理。行为证据可以是交易记录、决策记录、持仓计划、明确放弃某机会或延迟操作的记录。

## 下一阶段种子

系统不是为了多赚钱，首先是为了在极端情况下活下来。
