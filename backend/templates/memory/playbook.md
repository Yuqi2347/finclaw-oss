# 研究框架

本文件记录用户与 Agent 共同维护的投研方法论。它不是固定模板，后续可以根据用户认知迭代、删除、合并、改写或新增研究维度。

## 默认研究维度

### 业务理解
- 这家公司或资产靠什么赚钱？
- 核心收入来源、利润来源和增长来源是否清晰？
- 商业模式是否依赖一次性事件、周期景气或持续竞争力？

### 产业逻辑
- 所处行业处于什么周期和阶段？
- 驱动行业变化的核心变量是什么？
- 公司在产业链中的位置、议价能力和瓶颈价值是什么？

### 财务健康
- 盈利能力、现金流和资产负债结构是否支撑当前叙事？
- 增长质量是否稳定，是否依赖激进扩张或会计因素？
- 财务变化与业务逻辑是否相互印证？

### 估值位置
- 当前估值在历史区间和同行对比中处于什么位置？
- 当前价格隐含了什么增长预期和风险补偿？
- 如果核心假设低于预期，安全边际是否足够？

### 市场结构
- 当前市场关注的主线、资金偏好和交易拥挤度是什么？
- 标的逻辑是被市场充分定价、低估，还是处于分歧阶段？
- 短期价格变化背后是信息增量、情绪变化还是资金行为？

### 风险识别
- 什么情况下投资逻辑会失效？
- 市场当前最担心的风险是什么，是否已经反映在价格中？
- 是否存在被忽视的反身性风险、政策风险、技术替代或竞争格局变化？

## 使用原则

- 研究结论必须服务于清晰的问题，而不是堆砌资料。
- 事实、推理和判断要区分清楚。
- 当材料不足时，应明确不足，而不是用泛泛总结替代分析。
- 每次研究都可以反向修正本框架：如果用户明确形成新的研究共识，应更新本文件。
